# OWASP Top 10 for LLM Applications (2025年バージョン) - 日本語要約

このドキュメントは、OWASP Top 10 for Large Language Model (LLM) Applicationsの2025年バージョンの草案です。2023年に開始されたコミュニティ主導の取り組みで、AIアプリケーションに特有のセキュリティリスクを強調しています。

## 2025年の変更点
- **Unbounded Consumption**: サービス拒否（DoS）から拡張し、リソース管理と予期せぬコストのリスクを取り込む
- **Vector and Embeddings**: RAG（Retrieval-Augmented Generation）と埋め込み方法のセキュリティに焦点
- **System Prompt Leakage**: システムプロンプトが安全に隔離されていない場合の実世界での脆弱性
- **Excessive Agency**: エージェントアーキテクチャの増加によるリスク拡大

## 主な脆弱性と対策

### 1. LLM01:2025 プロンプトインジェクション
**説明**: ユーザーのプロンプトがLLMの振る舞いを意図せず変更するリスク。人間に見えない入力でもモデルの振る舞いを操作できる場合がある。

**リスク**:
- 機密情報漏洩
- 未承認アクセス
- 任意コマンド実行
- マルチモーダルAIにおけるクロスモーダル攻撃（画像やテキストの複合入力を利用した攻撃）

**対策**:
1. モデルの振る舞いを制限する（システムプロンプトの厳格なコンテキスト遵守）
2. 明確な出力フォーマットを定義し、フィルタリングを実装
3. 入力/出力フィルタリング（セマンティックフィルタと文字列チェック）
4. 高リスクアクションに対してヒューマンインザ Loop制御
5. 外部コンテンツの分離と識別
6. 定期的なペネトレーションテスト

### 2. LLM02:2025 機密情報漏洩
**説明**: 個人識別情報（PII）、財務詳細、医療記録などの機密情報が漏洩するリスク。

**対策**:
- データの厳格な制御とアクセス制限
- エラーメッセージを通じた機密情報漏洩防止

### 3. LLM03:2025 サプライチェーン脆弱性
**説明**: LLMのトレーニングデータ、モデル、デプロイメントプラットフォームにおけるサプライチェーンリスク。

**リスク**:
- バイアス出力
- セキュリティ侵害
- システム故障

**対策**:
1. データの洗浄と検証
2. 最小特権原則に基づく厳格なアクセス制御
3. フェデレーテッドラーニングの採用
4. ユーザー教育と透明性の向上
5. ホモモルフィック暗号化の使用

### 4. LLM04:2025 ベクトルと埋め込みの脆弱性
**説明**: RAGシステムにおけるベクトルと埋め込みのリスク。

**リスク**:
- 未承認アクセス
- データ漏洩
- 埋め込み逆転攻撃

**対策**:
1. 細粒度アクセス制御の実装
2. 信頼できるソースからのデータ検証
3. データの分類とタグ付け
4. リトリEEブ活動の詳細なロギング

### 5. LLM05:2025 誤情報
**説明**: LLMが生成する虚偽または誤解を招く情報（ハルシネーション）のリスク。

**対策**:
1. RAG（Retrieval-Augmented Generation）の採用
2. モデルの微調整とパラメータ効率的なチューニング
3. クロスバリデーションとヒューマンオーバーサイト
4. 自動検証メカニズムの実装
5. ユーザーインターフェース設計による責任ある使用の促進

### 6. LLM07:2025 システムプロンプト漏洩
**説明**: システムプロンプトが機密情報を含む場合のリスク。

**対策**:
1. 機密データとシステムプロンプトを分離する
2. ガードレールの実装（LLM外部での厳格な振る舞い制御）

### 7. LLM06:2025 過剰なエージェンシー
**説明**: LLMが過剰な機能性、権限、または自律性を持つ場合のリスク。

**対策**:
1. エクステンションとその機能の最小化
2. オープンエンドのエクステンションの回避
3. 最小限の権限でエクステンションを実行する
4. 高影響アクションに対してユーザー承認を要求

### 8. LLM10:2025 無制限消費
**説明**: LLMアプリケーションが無制限な推論を許可するリスク。

**対策**:
1. 入力検証とサイズ制限
2. レート制限とユーザークォータの実装
3. リソース割り当ての動的管理
4. サンドボックス技術によるリソースアクセス制限
5. ウオーターマーキングの採用
